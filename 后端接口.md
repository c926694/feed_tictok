# 后端接口总结

本文档基于当前代码中的路由与控制器整理，适用于联调与接口自查。

## 1. 通用约定

- 基础返回结构：

```json
{
  "success": true,
  "message": "ok",
  "data": {}
}
```

- 鉴权方式：`Authorization: Bearer <token>`
- 除 `users/login`、`users/register` 外，其它接口均需 JWT 鉴权。
- 推荐网关前缀：`/api`（前端 axios `baseURL` 为 `/api`）

## 2. 用户模块（`/users`）

### 2.1 注册

- 方法与路径：`POST /users/register`
- 鉴权：否
- 入参（JSON）：

```json
{
  "username": "test",
  "password": "123456",
  "re_password": "123456"
}
```

- 返回：`data` 为 `user_id`

### 2.2 登录

- 方法与路径：`POST /users/login`
- 鉴权：否
- 入参（JSON）：

```json
{
  "username": "test",
  "password": "123456"
}
```

- 返回：`data` 为 `token` 字符串

### 2.3 获取当前用户信息

- 方法与路径：`GET /users/me`
- 鉴权：是
- 返回 `data`（`UserInfoRes`）字段：
  - `user_id`
  - `username`
  - `nickname`
  - `avatar_URL`
  - `follow_count`
  - `follower_count`
  - `video_count`

### 2.4 修改个人信息

- 方法与路径：`POST /users/me`
- 鉴权：是
- Content-Type：`multipart/form-data`
- 入参：
  - `nickname`（可选）
  - `avatar`（可选，文件）
- 返回：更新后的 `UserInfoRes`

### 2.5 退出登录

- 方法与路径：`DELETE /users/logout`
- 鉴权：是
- 返回：`data` 通常为 `null`

## 3. 视频模块（`/videos`）

### 3.1 发布视频

- 方法与路径：`POST /videos/create`
- 鉴权：是
- Content-Type：`multipart/form-data`
- 入参：
  - `title`（文本）
  - `description`（文本）
  - `cover`（封面文件）
  - `play`（视频文件）
- 返回 `data`（`VideoRes`）：
  - `id`
  - `url`

### 3.2 删除视频

- 方法与路径：`DELETE /videos/:id`
- 鉴权：是
- 说明：仅作者可删；会删除视频及关联评论，并清理对应点赞集合与 MQ 文件删除事件。

### 3.3 查询我的视频

- 方法与路径：`GET /videos/me?limit=60`
- 鉴权：是
- Query：
  - `limit`（可选，默认 `60`）
- 返回：`VideoInfoRes[]`

### 3.4 查询视频详情

- 方法与路径：`GET /videos/:id`
- 鉴权：是
- 返回：`VideoInfoRes`

### 3.5 普通 Feed 流

- 方法与路径：`GET /videos/feed?limit=3&last_score=...`
- 鉴权：是
- Query：
  - `limit`（可选，默认 `3`）
  - `last_score`（可选，分页游标）
- 返回 `FeedVideoRes`：
  - `feed_video_list`：`VideoInfoRes[]`
  - `last_score`：下一页游标

### 3.6 热榜 Feed 流（按热度 score）

- 方法与路径：`GET /videos/feed/hot?limit=3&last_score=...`
- 鉴权：是
- Query：
  - `limit`（可选，默认 `3`）
  - `last_score`（可选，热度 score 游标）
- 返回 `FeedVideoRes`：
  - `feed_video_list`：`VideoInfoRes[]`（按热度顺序）
  - `last_score`：下一页热度游标

### 3.7 关注 Feed 流

- 方法与路径：`GET /videos/feed/follow?limit=3&last_score=...`
- 鉴权：是
- Query：
  - `limit`（可选，默认 `3`）
  - `last_score`（可选，基于发布时间的游标）
- 返回 `FeedVideoRes`

### 3.8 `VideoInfoRes` 字段

- `id`
- `author_id`
- `author_name`
- `author_avatar`
- `title`
- `description`
- `coverURL`
- `playURL`
- `comment_count`
- `like_count`
- `is_liked`
- `is_follow`
- `created_at`

## 4. 点赞模块（`/likes`）

### 4.1 切换视频点赞

- 方法与路径：`POST /likes/video/switchLike/:id`
- 鉴权：是
- 返回 `LikeVideoRes`：
  - `video_id`
  - `is_liked`（当前状态）

### 4.2 切换评论点赞

- 方法与路径：`POST /likes/comment/switchLike/:id`
- 鉴权：是
- 返回 `LikeCommentRes`：
  - `comment_id`
  - `is_liked`

## 5. 关注模块（`/follows`）

### 5.1 切换关注

- 方法与路径：`POST /follows/switchFollow/:follower`
- 鉴权：是
- 说明：`:follower` 是目标用户 ID
- 返回 `FollowRes`：
  - `following`（目标用户 ID）
  - `is_follow`（当前状态）

## 6. 评论模块（`/comments`）

### 6.1 发表评论

- 方法与路径：`POST /comments`
- 鉴权：是
- 入参（JSON）：

```json
{
  "video_id": 1,
  "content": "评论内容"
}
```

- 返回：`CommentRes`

### 6.2 删除评论

- 方法与路径：`DELETE /comments/:id`
- 鉴权：是
- 说明：仅评论作者可删

### 6.3 评论列表

- 方法与路径：`GET /comments/list/:videoId`
- 鉴权：是
- 返回：`CommentRes[]`

### 6.4 `CommentRes` 字段

- `id`
- `commenter`
- `video_id`
- `content`
- `like_count`
- `is_liked`
- `author`（`UserInfoRes`）
- `created_at`

## 7. 热榜分数异步更新（MQ）

以下动作会触发视频热度异步更新：

- 视频点赞/取消点赞
- 评论新增/删除

异步流程：

- 业务写库后投递 `video.hot` 事件
- `VideoHotConsumer` 消费后重新计算热度：
  - `HotScore = like_count * 2 + comment_count + video_id / 1e10`
- 更新 Redis 热榜 ZSet：`feed:hot:video`

## 8. 联调建议

- 热榜分页请使用后端返回的 `last_score` 原样透传。
- 热榜排序以后端返回顺序为准，前端不做二次排序。
- 前端请求建议携带时间戳参数避免中间层缓存旧数据。
